I have a LARGE text file and had to do something like
"grep -e foo -e bar -e baz < LARGE_FILE > SOME_LINES ; grep -v -e foo -e bar -e baz < LARGE_FILE > THE_OTHER_LINES

Since LARGE is very, very large file, it takes at least 10 minutes to run it through grep once, so I would like to avoid the other grep.

It would be nice if grep could just give me the two output files in one pass of the file.

So the idea is: shepsalgreppy -e some -e patterns -o output1 -v -e other_pattern -e and_some_more -o output2 > the_rest_of_the_lines

cat LARGE_FILE | shepsalgreppy ...   <--- HOW THIS WILL BE USED

do
{
    char * line = read_line_from_stdin();

    if (! line)
    {
        break;
    }
} while true;


See "man grep" for the meaning of "-e", "-i", "-v", and "-w". 
The "-o" is just my idea for specifying the output file for a block of patterns. 
The normal grep has a "-o" option, so maybe use "-O" instead (not used by grep), if we want to support grep's "-o" option. 
Personally I don't think I have ever used grep's "-o" option.
