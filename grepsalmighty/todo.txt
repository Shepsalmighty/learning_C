About libpcre2...

The library can work with either 8, 16, or 32 bits per code-point. For normal ASCII text, 8 bits per code-point seems to work in my very limited tests.
Maybe 16 or 32 bits per code-point is needed, if you want to use Unicode text.

So #define OCE2_CODE_UNIT_WIDTH 8 before you #include <pcre2.h>

Regular expressions is kind of like a small program, so the first thing you need to do is to call pcre2_compile().
You only need to do that once per regular expression.

In regular expression, you often "match all of this text", but only actually need the "this" part. 
The rest is just to make sure you get the correct "this". You do that by using something like "match all of (this) text".
libpcre2 will return various matches (in this case "match all of this text" and "this") by giving you a set of start and end offsets for each match.
To allocate memory for those offset, you call pcre2_match_data_create_from_pattern() on the compiled regular expression from the previous step.
You only need to do that once per compiled regular expression.

And finally you call pcre2_match() to check if a compiled regular expression match a given text.
You can do that multiple times with a given compiled regular expression and a given text 
(with different start offsets to get the next match in the text)
or with a given compiled regular expression and different text to see which texts match the compiled regular expression and which don't.
(Sounds familiar?)

In our case we only want to know if a given line of text match a regular expression or not.
We don't really care where the match is in the text. If we did care about that,
we would call pcre2_get_ovector_pointer() and then use ovector[ 2 * i + 0 ] and ovector[ 2 * i + 1 ] to get the start and end offset
of match number i. (The return value of pcre2_match() is the number of matches.)

Finally when we're all done with the compiled regular expression from pcre2_compile() and 
the match data from pcre2_match_data_create_from_pattern(), we free using pcre2_match_data_free() and pcre2_code_free().

And I think that's it...
